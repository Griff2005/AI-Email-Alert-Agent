Overview

Email Alert Triage Agent

An AI-powered compliance case management system for elevator KPI alerts. Classifies inbound alert emails, extracts structured fields, creates or updates compliance cases, generates professional outbound follow-up emails, and tracks deadlines with automatic escalation.

The AI brain is Claude Haiku, invoked via the claude --print CLI in headless mode — no direct SDK dependency required.

Quick Start

Quick Start

1. Install dependencies

cd "AI Email Alert Agent"
pip install -r requirements.txt

2. Configure credentials

cp .env.example .env
# Edit .env — fill in AGENT_EMAIL, AGENT_EMAIL_PASSWORD,
# IMAP_HOST, SMTP_HOST, and DEMO_RECIPIENT_EMAIL

3. Run the demo (no real inbox required)

python3 src/agent.py demo

4. Start the full agent + web UI

python3 src/agent.py run
# Open http://localhost:5000

CLI entry point. Parses arguments and dispatches to one of four commands: ingest, demo, run, reply.

Load and return the sample email list from data/sample_emails.json.

Returns: List of email dicts with keys: id, subject, from, to, date, body.
Raises:
  • SystemExit — If the file is not found.
def _store_email (em: dict) → str

Insert a sample email dict into the database and return its email_id. Sanitises the body via sanitize_email_content before storing.

ArgTypeDescription
emdictEmail dict with at least subject, from, to, date, body.
Returns: The email_id string used for the inserted record.
def cmd_ingest (args) → None

Process all sample emails from data/sample_emails.json. Safe to run multiple times — duplicates are ignored.

def cmd_demo (args) → None

Run the demo: ingest all sample emails and print a formatted results table with case UUIDs for use with the reply command.

def cmd_run (args) → None

Start the full agent: initialise schema → start APScheduler → start IMAP polling thread (if configured) → start Flask (blocking). Passes use_reloader=False to prevent Werkzeug from forking the process and doubling the scheduler.

def cmd_reply (args) → None

Interactive CLI reply handler. Prompts for reply content terminated by ---END---, calls case_manager.process_reply, and if resolution is indicated, asks the user to confirm before closing. Cases are never closed without explicit confirmation.

ArgTypeDescription
argsNamespaceMust include case_id attribute.

Module

database.py

SQLite schema creation and all query helpers. Thread-safe via thread-local connections and a module-level write lock.

defget_connection() → sqlite3.Connection

Return a per-thread SQLite connection, creating it on first call. Uses threading.local so Flask, APScheduler, and the IMAP thread each get their own connection. Configures WAL mode for concurrent reads and PRAGMA foreign_keys=ON.

Returns: Open connection with row_factory = sqlite3.Row.
def_execute_write(sql: str, params: tuple) → Cursor

Execute a write statement under the module-level write lock. Serialises all INSERT/UPDATE/DELETE across threads — SQLite supports only one concurrent writer.

ArgTypeDescription
sqlstrINSERT, UPDATE, or DELETE statement.
paramstupleBind parameters.
definit_schema() → None

Create all 7 tables and 5 indexes if they do not exist. Safe to call on every startup.

definsert_email(...) → None

Insert an inbound KPI alert email. Uses INSERT OR IGNORE on message_id — safe to call twice with the same email.

defget_case_by_grouping_key(grouping_key: str) → Row | None

The deduplication gate. Returns an existing case if one exists for this building/device/period combination, or None to create a new case.

definsert_case(...) → None

Insert a new compliance case. Sets status to 'open' and both timestamps to current UTC. grouping_key has a UNIQUE constraint.

definsert_case_event(...) → None

Append an immutable event to a case's audit trail. Events are never updated or deleted. Every state change is recorded here.

defupsert_followup(followup_id, case_id, deadline) → None

Insert a follow-up deadline record. INSERT OR IGNORE on case_id (UNIQUE) — re-processing an email never resets the follow-up counter.

defget_overdue_followups() → List[Row]

Return all follow-up records whose deadline has passed and whose case is still open. Ordered by deadline ascending (most overdue first).

defincrement_followup_count(case_id: str) → int

Thread-safe read-modify-write. Increments follow_count and records last_check.

Returns: New follow_count value.
Module

claude_client.py

Claude CLI subprocess wrapper. All AI calls in the project go through this module. Includes SHA-256 response cache and two-layer prompt injection defence.

defdetect_injection(text: str) → bool

Scan text for 8 prompt-injection regex patterns. Called at layer 1 (inbound email content) and layer 2 (Claude's own response) as a two-layer defence.

Returns: True if any pattern matches.
defsanitize_email_content(raw: str) → str

Strip HTML → collapse whitespace → wrap in --- EMAIL CONTENT START/END --- delimiters. Makes the data boundary explicit in every Claude prompt.

Returns: Sanitised plain-text string ready to embed in a prompt.
defcall_claude(prompt: str, use_cache: bool = True) → str

Invoke claude --print --model <CLAUDE_MODEL> via subprocess with prompt on stdin. Checks the SHA-256-keyed JSON cache first; stores the response on a miss. Also scans Claude's output for injection-like language (layer 2).

ArgTypeDescription
promptstrFull prompt string passed via stdin.
use_cacheboolCheck/store in on-disk cache (default True).
Returns: Claude's response as a stripped string.
Raises:
  • FileNotFoundError — claude not found on PATH.
  • RuntimeError — Non-zero CLI exit code.
defcall_claude_json(prompt: str, use_cache: bool = True) → dict

Wraps call_claude. Strips markdown code fences before parsing so callers always receive a clean dict.

Raises:
  • ValueError — Response is not valid JSON after stripping fences.
Module

classifier.py

Email classification via Claude CLI. Two-stage: fast keyword pre-filter, then Claude for precise classification with confidence scoring.

defquick_filter(subject: str) → bool

Case-insensitive keyword check against the subject line only. Prevents unnecessary Claude calls for non-KPI emails (out-of-office replies, spam).

Returns: True if a KPI trigger keyword is found; False to skip.
defclassify_email(subject: str, body: str) → dict

Classify an email into one of 6 KPI case types using Claude. Runs injection detection on subject and body before the Claude call. Coerces unrecognised types to 'UNKNOWN'; clamps confidence to [0.0, 1.0].

Returns: Dict with case_type, confidence, reasoning, injection_detected.
Module

extractor.py

Field extraction, grouping key generation, and outbound email body generation — all via Claude CLI.

defextract_fields(subject, body, case_type) → dict

Prompt Claude to extract up to 12 structured fields from a KPI alert email. All values are either a non-empty string or Python None.

Returns: Dict with keys: building, device, contractor, due_date, scheduled_date, period, hours_required, hours_actual, description, last_activity_date, elapsed_days, directive_tasks.
defgenerate_grouping_key(case_type, building, device, period) → str

Deterministic dedup key. Two emails for the same scenario produce the same key despite minor formatting differences. Format: {case_type}|{building}|{device}|{period} (all components normalised to lowercase, collapsed whitespace, empty string for None).

Returns: Pipe-delimited normalised string. Example: cat1_compliance|123 example road|b-4 #731842|
defgenerate_email_body(case_type, fields, case_id) → str

Claude writes a professional plain-text outbound email body (max 200 words, 5-day deadline). Always called with use_cache=False so each case gets a fresh email.

Returns: Plain-text email body string.
Module

case_manager.py

Central pipeline orchestrator. Coordinates classify → extract → deduplicate → store → notify for every inbound email.

defprocess_email(email_id, subject, body, ...) → dict

Seven-step pipeline: quick_filter → classify → route low-confidence → extract_fields → grouping_key → create/update case → send outbound email.

Returns: Dict with action ('created'/'updated'/'skipped'/'review_flagged'), case_id, case_type, grouping_key, injection_detected.
def_create_new_case(...) → None

Insert case row → store extracted fields → schedule follow-up deadline → generate and send outbound email.

def_update_existing_case(...) → None

Append email_received event and refresh any non-null fields from the new email. No new outbound email — the case is already in progress.

defprocess_reply(case_id, reply_text, verbose) → dict

Analyse a pasted reply with Claude. Appends a reply_received event. Cases are never auto-closed — only explicit human action can set status='closed'.

Returns: Dict with analysis, satisfies_action, flagged_for_review, event_id.
Raises:
  • ValueError — If case_id not found.

Database Schema

Seven SQLite tables with five performance indexes.

emails

Every inbound KPI alert email received. INSERT OR IGNORE on message_id prevents duplicates.

ColumnTypePurpose
email_idTEXT PKApplication UUID.
message_idTEXT UNIQUERFC 2822 Message-ID header. Dedup key.
subjectTEXTDecoded subject line.
raw_bodyTEXTOriginal body (may contain HTML).
normalized_textTEXTHTML-stripped, whitespace-normalised body.
processedINTEGER0 until case pipeline completes; then 1.

cases

One row per active compliance case. grouping_key is UNIQUE — the deduplication gate.

ColumnTypePurpose
case_idTEXT PKApplication UUID.
case_typeTEXTOne of 6 KPI case type constants.
statusTEXTopen or closed.
priorityTEXTlow / medium / high / critical.
grouping_keyTEXT UNIQUENormalised dedup key.
building / device / contractorTEXTExtracted compliance fields.
due_date / periodTEXTExtracted dates.

case_events

Immutable audit log. Every state change appends a row — nothing is ever deleted.

extracted_fields

Raw extraction data separate from the case row — each field with its source email and confidence score.

outbound_messages

Every drafted/sent email. Stores both intended_to (production audit) and actual_to (always demo address in DEMO_MODE).

followups

One record per open case. Tracks deadline, last check, and follow count.

manual_reviews

Cases requiring human attention: low confidence, injection detected, reply suggests resolution, escalation threshold reached.

Security Model

Prompt Injection Prevention

Every piece of email content passes through sanitize_email_content() before being embedded in a Claude prompt:

  1. HTML stripping — BeautifulSoup removes all tags.
  2. Whitespace normalisation — collapses spaces/tabs, removes blank lines.
  3. Delimiter wrapping — content enclosed in --- EMAIL CONTENT START --- / --- EMAIL CONTENT END ---.
  4. Inbound scan (layer 1) — detect_injection() checks body and subject before the Claude call.
  5. Output scan (layer 2) — Claude's response is also scanned for override language.

Demo Recipient Enforcement

In DEMO_MODE=true, the actual_to override in create_draft() is unconditional. No calling code can bypass it. The intended production address is stored in intended_to for audit only.

No Auto-Closure

Cases can only be closed by explicit human action: the reply CLI prompt or the web UI "Close Case" button. No automated process — including email content, reply analysis, or follow-up generation — can close a case.
